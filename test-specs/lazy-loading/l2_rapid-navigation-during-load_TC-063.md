## Network Behavior
- `GET /api/v2/elitea_core/tools/prompt_lib/21?query=&sort_by=created_at&sort_order=desc&limit=20&offset=0` — the Toolkits list endpoint. Response shape: `{ "rows": [...], "total": 6 }`. Unlike Agents/Pipelines' `applications/prompt_lib` endpoint, this account's current volume (6) fits entirely within the default `limit=20` — confirmed **no second page ever fires**, with or without scrolling. Observed duration: 1401-2120ms across repeated live runs — this is the window a "navigate away before load completes" test needs to interrupt; a fixed 2-second wait sits at/past this boundary on this account's current volume, so navigating away with **zero** wait is the reliable way to land inside the fetch's in-flight window.
- On navigating away mid-fetch (`page.goto()` to a new route), the FULL page — not just the list panel — tears down and re-bootstraps on the destination route. Confirmed live: 8 distinct in-flight `/api/v2/**` + `socket.io` requests (not just the toolkits list itself) all resolved as `net::ERR_ABORTED` in a single interrupted navigation: `tools/prompt_lib`, `toolkits/prompt_lib` (the toolkit-type schema catalog, see below), `toolkit_types/prompt_lib`, `tags/prompt_lib`, `social/authors`, `secrets/secrets`, plus two `socket.io` polling round-trips. Zero of these aborts produced a console error or warning across 3 live trials.
- `GET /api/v2/elitea_core/toolkits/prompt_lib/{ownerId}` — a **different, unrelated** endpoint despite the similar name: returns a large (~700KB) static catalog of all possible toolkit *type* schemas (used by the "New Toolkit" creation wizard at `/app/toolkits/create`), not the user's configured toolkit instances. Do not confuse this with `tools/prompt_lib` above, which is the actual list this case is about. Observed once as a `404` with no `{ownerId}` path segment during a transient login-race redirect to `/app/toolkits/create` (see Known Defects) — not reproduced on any stable, direct navigation to `/app/toolkits/all`.
- `GET /api/v2/elitea_core/toolkit_types/prompt_lib/{ownerId}` — populates the "Types" filter chip row (Artifact/Browser/GitHub/Jira/Pandas).
